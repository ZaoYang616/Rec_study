# 项目实践时发现的“跷跷板”与“负迁移”现象问题及其解决方案

## 一、 业务背景与问题的爆发

在我的电商搜广推多目标学习（MTL）项目中，核心任务是基于经典的 Ali-CCP 数据集，同时预测用户的**点击率 (CTR)** 和 **转化率 (CVR)**。

然而，该业务场景存在极其严峻的数据痛点——**极端的漏斗效应与样本不平衡**。全局点击正样本（CTR）占比约 4%，而真实的购买正样本（CVR）仅有极端的 0.03%（万分之三）。

在我将模型从朴素双塔升级，并引入 DIN（目标注意力）和 AFM（特征自动交叉）之后（即项目 V4/V5 阶段），模型表现出了极其典型的 **“跷跷板现象”** 和 **“负迁移现象”**：
- **表象观测**：在训练过程中，CTR 任务的 AUC 稳步上涨（最高达到 0.567+），但 CVR 任务的 AUC 却不升反降（从 0.566 跌回 0.555）。两个目标互相拉扯，按下一个，翘起另一个。
---

## 二、 破局之路：我的四次核心架构与工程迭代


### 迭代 1：物理切断梯度(`.detach()`)
我首先在 CTR 塔流向 CVR 塔的特征枢纽处，引入了 `shared_hidden.detach()` 操作。
- **目的与效果**：这是防止负迁移最立竿见影的手段。它强制切断了 CVR 塔那极其稀疏、带有强烈购买偏差的梯度回传。底座被彻底交由数据量庞大的 CTR 任务主导更新，CVR 任务对底座特征变为“只读不写”，防止了底层参数被破坏。

### 迭代 2：过滤噪音的桥接清洗(Bridge Gate)
虽然切断了梯度，但底座传来的特征依然带有“点击偏见”。为此，我在两塔之间搭建了一座桥梁——`Bridge Gate NN`。
- **目的与效果**：我利用极其代表个人消费习惯的“人设向量”（`persona_emb` = 用户画像 + 动态行为），生成了一组动态缩放系数。在底座特征进入 CVR 塔之前，对其进行了一次大清洗：压制那些只和“标题党点击”有关的维度，放大和该用户“真实购买意图”相关的维度。

### 迭代 3：用户特征全程参与(引入 PEPNet)
为了彻底解决底座特征被“过分抽象化”的问题，我将普通的 CVR 多层感知机升级为了 **PEPNet (参数个性化转化塔)**。
- **目的与效果**：我将用户的原始具象特征 `persona_emb` 直接输入到 CVR 塔的每一层（`PPNetLayer`）中，用于动态生成和修改该层的权重。CVR 塔不再只能被动接受底层传来的模糊特征，而是每一层都能重新“看一眼”用户最清晰的原始画像和历史兴趣，实现了千人千面的参数定制。

### 迭代 4：消灭梯度震荡 (Focal Loss)
在早期的尝试中，为了唤醒 CVR，我曾简单粗暴地给 CVR 正样本施加了 500 倍的权重惩罚，这导致了极其剧烈的梯度震荡（Loss 时常在 0.07 到 1.6 之间坐过山车）。
- **目的与效果**：在最终版本中，我彻底废弃了硬核惩罚，全面换用 **Focal Loss**。我赋予 CVR 极高的 `alpha=0.95` 平衡权重，并通过 `gamma=2.0` 强迫模型忽略大量容易猜对的负样本，死死盯住那万分之三的购买正样本。这使得双任务的 Loss 极其平滑，彻底稳住了训练阵脚。

---

## 三、 解决效果与最终成果

通过上述 **“截断 + 门控过滤 + 特征刺透 + 损失平衡”** 的四步走战略，我的模型迎来了脱胎换骨的蜕变（V6）：

1. **彻底抹平震荡**：训练 Loss 不再出现“过山车”，CTR 和 CVR 的 Loss 均以极其平滑的曲线稳步下降。
2. **打破跷跷板瓶颈**：在测试集评估中，CTR 和 CVR 的 AUC 不再互相打架，而是携手稳步攀升，最终**双目标的 AUC 均突破了 0.56+ 的工业级基准线**。
3. **架构健壮性验证**：这一系列改造证明了，在极度稀疏的业务场景下，单纯的特征相加是无效的，必须引入细粒度的门控机制与防污染隔离。

## 四、 下一步规划：通往 MMoE 的终极解法

尽管目前的 PEPNet 架构取得了显著成功，但 Shared-Bottom 这个架构本身就容易发生负迁移。

**下一步计划：**
引入 **MMoE (Multi-gate Mixture-of-Experts，多门控混合专家网络)**，并为 CTR 和 CVR 分别配备独立的门控（Gates）去动态挑选专家的意见。
